macro_rules! log_post_fatal {
    ($category:expr, $object:expr, $msg:expr) => { ... };

}
Expand description

log_post_fatal! emits the same target-prefixed Fatal:<class>:<object> line a Perl Fatal('class', 'object', …) would, and then converts to a PostError::Processing so the calling function can early-return via ? like Perl’s die.

The logger’s “Fatal:” prefix is matched by latexml_core::util::logger::log line 82 (starts_with("Fatal:") → no severity prefix added, the class-prefix Fatal: is the severity).
