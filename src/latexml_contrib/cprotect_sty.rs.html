1//! `cprotect.sty` — verbatim material inside macro arguments.
2//!
3//! There is **no upstream Perl binding** for this package (Perl LaTeXML 0.8.8
4//! reports `Warning:missing_file:cprotect` and then `Error:undefined:\cprotect`
5//! on the issue #347 witness), so this is a beyond-Perl contribution and lives
6//! in `latexml_contrib`.
7//!
8//! ## What the package does, and how the port differs
9//!
10//! `cprotect.sty` (Bruno Le Floch, TL `tex/latex/cprotect/cprotect.sty`) works
11//! around a pure-TeX limitation: a braced macro argument is tokenized *before*
12//! the macro runs, so a `\verb` inside it never gets to change catcodes and the
13//! `\`, `%`, `#`, … of its body have already been mis-tokenized. Its machinery
14//! (`\CPT@readContent` / `\makeallother` / `\CPT@Write`) reads the argument with
15//! every catcode forced to 12, **writes it out to `\jobname-N.cpt`**, and hands
16//! the command `\protect\input <that file>` instead — the re-`\input` is what
17//! re-tokenizes the text so `\verb` can finally do its job.
18//!
19//! LaTeXML has a mouth-level equivalent of "write it out and read it back"
20//! that needs no disk round-trip: eTeX's `\scantokens` (`latexml_engine::etex`,
21//! Perl `eTeX.pool.ltxml` L251) opens a fresh `Mouth` over a string. So the
22//! port keeps cprotect's *reading* half verbatim and replaces its *file* half:
23//!
24//! ```text
25//! \cprotect\section{<raw>}   ==>   \section{\scantokens{<raw>}}
26//! ```
27//!
28//! `<raw>` is read with the specials neutralized (the `\makeallother` stand-in),
29//! except that `{`/`}` keep catcodes 1/2 so `read_balanced` does the brace
30//! counting that `\CPT@gobbleOneB` does by hand.
31//!
32//! Not ported: `\ReadVerbatimUntil` (the package-author-level generic verbatim
33//! reader) and the `gobbling-escape=`/`gobbling-letter=` options — the latter
34//! only tune the `^^E`/`^^L` sentinels of the file round-trip we do not perform.
35//!
36//! ## One deliberate deviation from `cprotect.sty`
37//!
38//! A **multi-letter signature keeps going after a single-token argument.** In
39//! the package, `\CPT@read@mone` (the `m` branch for an argument that is not a
40//! `{`-group) is `\CPT@cs\CPT@next` — it emits the command and stops, dropping
41//! the rest of the signature. So `\cprotect[mm]\section*{\verb|\x|}` protects
42//! nothing and errors in pdflatex (`! LaTeX Error: \verb illegal in argument`,
43//! verified TL2025), the same as the plain `\cprotect\section*{…}` that the
44//! default `om` signature cannot express either. Here the loop continues, so
45//! `[mm]` is the working spelling for a starred sectioning command. This only
46//! ever *adds* a case the package errors on — every signature it handles
47//! correctly is handled identically. Plain `\cprotect\section*{…}` still
48//! degrades exactly as the package does (the `*` is the `m` argument, the group
49//! reaches `\section*` unprotected), so a document that works with pdflatex
50//! converts the same way here.
51//!
52//! ## Two known limitations, both on input pdflatex also rejects
53//!
54//! **1. An argument its command reads verbatim keeps the `%` sentinel.**
55//! `\cprotect[mm]\href{http://x.org/a_b}{link \verb|\eta|}` yields
56//! `href="http://x.org/a_b%"`: the URL is consumed under semiverbatim catcodes,
57//! where `%` is catcode 12, so the end-of-line sentinel survives as a character.
58//! Anything we emit is at the mercy of a consumer that reads literally, and the
59//! sentinel has no character that is a comment under BOTH catcode regimes — `%`
60//! is precisely the one semiverbatim neutralizes. Two fixes were measured and
61//! rejected as worse: dropping the sentinel when the argument carries no escape
62//! trades the `%` for the mouth's end-of-line, which HTML collapses at the end
63//! of a title but renders as a **visible** space mid-flow
64//! (`(a[b]c : …)` for `\cprotect\cmd[a[b]c]{…}`); and skipping the round trip
65//! altogether emits the neutralized catcode-12 space as a literal glyph
66//! (`short ̵toc`). pdflatex + cprotect cannot compile `\cprotect\href` at all
67//! (`TeX capacity exceeded`, no PDF), so there is no reference behaviour being
68//! missed — protecting a URL/filename/citekey argument is outside the package's
69//! own domain.
70//!
71//! **2. A pre-tokenized argument gains a space inside `\verb`.** When
72//! `\cprotect` is reached from inside another macro's argument
73//! (`\wrap{\cprotect\emph{\verb|\zeta|}}`) the group has already been tokenized,
74//! so the raw read yields real CS tokens and `writable_tokens` spaces every
75//! control word: the body comes back as `\zeta ` . Protection is impossible
76//! there in principle — the mis-tokenization happened at `\wrap`'s argument read
77//! — and the result is still far better than the unprotected `<verbatim/>` plus
78//! italic `ζ`. In the package this input is a hard error, since `\cprotect` is
79//! `\outer` and therefore illegal in any macro argument
80//! (`! Forbidden control sequence found while scanning use of \wrap`); this
81//! binding carries no `\outer` restriction, so it accepts more than TeX does.

85/// cprotect's `\makeallother` stand-in: the characters whose catcode must be
86/// neutralized while the protected argument is read, so that the text survives
87/// to the `\scantokens` re-read intact. `{`/`}` are deliberately NOT in the
88/// list — keeping them as BEGIN/END lets `read_balanced` find the end of the
89/// argument, which is exactly what cprotect emulates by counting its own
90/// `\CPT@other@bgroup`/`\CPT@other@egroup` in `\CPT@gobbleOneB`.
91///
92/// The space is included (cprotect makes it catcode 12 too): a run of spaces
93/// inside `\verb|a  b|` must not collapse before the re-read.
94///
95/// So is `\r`, the `\endlinechar` the mouth appends to every line. At its usual
96/// catcode 5 an end-of-line collapses to one space token and the argument comes
97/// back as a single line — which silently flattens the package's *other*
98/// headline case, `\cprotect\footnote{\begin{verbatim}…\end{verbatim}}`, into
99/// one run-on line. As catcode 12 it survives into the `\scantokens` string,
100/// where `Mouth::split_lines` breaks on it again. cprotect gets the same effect
101/// from `\newlinechar`\``\^^M` in `\CPT@Write`.
102static CPROTECT_OTHER: [char; 10] = ['\\', '%', '#', '$', '&', '^', '_', '~', ' ', '\r'];
103
104/// Open a group in which [`CPROTECT_OTHER`] is neutralized; the caller pops it.
105fn begin_protected_read() {
106  push_frame();
107  for special in CPROTECT_OTHER {
108    assign_catcode(special, Catcode::OTHER, Some(Scope::Local));
109  }
110}
111
112/// Read one balanced group whose opening `{` has already been consumed, with
113/// [`CPROTECT_OTHER`] neutralized. Mirrors `\CPT@readContent` under
114/// `\makeallother`.
115fn read_protected_group() -> Result<Tokens> {
116  begin_protected_read();
117  let raw = read_balanced(ExpansionLevel::Off, false, false);
118  pop_frame()?;
119  raw
120}
121
122/// The `[...]`-delimited twin of [`read_protected_group`] (`\CPT@read@d` with
123/// `#1#2` = `[`/`]`): peek for `open`, and when it is there read up to the
124/// MATCHING `close` with [`CPROTECT_OTHER`] neutralized. Returns `None` (having
125/// consumed nothing) when the optional argument is absent — `\CPT@read@d@none`.
126///
127/// The depth count is not decoration: `read_until` would stop at the first
128/// `close`, whereas cprotect explicitly balances its delimiter pair in
129/// `\CPT@gobbleOneB`, so `\cprotect\cmd[a[b]c]{…}` keeps its whole optional.
130fn read_optional_protected(open: Token, close: Token) -> Result<Option<Tokens>> {
131  let Some(token) = read_non_space()? else {
132    return Ok(None);
133  };
134  if token != open {
135    unread_one(token);
136    return Ok(None);
137  }
138  begin_protected_read();
139  // Scan inside a closure so the catcode frame is popped on the error path too.
140  let scan = (|| -> Result<(Vec<Token>, bool)> {
141    let mut raw: Vec<Token> = Vec::new();
142    let mut depth = 1_usize;
143    while let Some(token) = read_token()? {
144      if token == close {
145        depth -= 1;
146        if depth == 0 {
147          return Ok((raw, true));
148        }
149      } else if token == open {
150        depth += 1;
151      }
152      raw.push(token);
153    }
154    Ok((raw, false))
155  })();
156  pop_frame()?;
157  let (raw, closed) = scan?;
158  if !closed {
159    Error!(
160      "expected",
161      "delimiter",
162      s!("Runaway \\cprotect optional argument, looking for {close}")
163    );
164  }
165  Ok(Some(Tokens::new(raw)))
166}

168/// Wrap already-read raw tokens as `{\scantokens{<raw>%}}` — the port's stand-in
169/// for cprotect's `{\protect\input \jobname-N.cpt\relax}` (`\CPT@read@mbeg`).
170///
171/// The trailing `%` is cprotect's `\CPT@postText` in LaTeXML dress. Re-reading
172/// text through a mouth (`\input` there, `\scantokens` here) appends the
173/// end-of-line, i.e. a spurious space at the end of every protected argument;
174/// cprotect eats it by ending the written file with the `^^E^^L` control word
175/// (a CS name absorbs the following space, and the CS itself is `\def`'d
176/// empty). A comment character does the same job with the machinery we have.
177fn scantokens_group(raw: Tokens) -> Vec<Token> {
178  let mut out = vec![T_BEGIN!(), T_CS!("\\scantokens"), T_BEGIN!()];
179  out.extend(raw.unlist());
180  out.push(T_OTHER!("%"));
181  out.push(T_END!());
182  out.push(T_END!());
183  out
184}

186LoadDefinitions!({
187  // `\outer\long\def\cprotect{\icprotect}` + `\newcommand{\icprotect}[2][om]`
188  // (cprotect.sty, `\icprotect` / `\CPT@read@args`): an OPTIONAL argument
189  // signature, then the command token itself, then that command's own arguments
190  // read per the signature.
191  //
192  // The optional is read here rather than declared as a `[om]` parameter so the
193  // whole read stays in one place: the signature decides how many further
194  // arguments to consume and each of them needs the neutralized-catcode read.
195  DefMacro!("\\cprotect", {
196    let argsig = read_optional(None)?
197      .map(|sig| sig.to_string())
198      // `\icprotect`'s `[om]` default: one optional, then one mandatory.
199      .unwrap_or_else(|| "om".to_string());
200    let Some(command) = read_non_space()? else {
201      return Ok(Tokens!());
202    };
203    let mut out = vec![command];
204    for arg_kind in argsig.chars() {
205      match arg_kind {
206        // `\CPT@read@o` = `\CPT@read@d[]`: protect a `[...]` when present,
207        // otherwise consume nothing.
208        'o' => {
209          if let Some(raw) = read_optional_protected(T_OTHER!("["), T_OTHER!("]"))? {
210            out.push(T_OTHER!("["));
211            out.extend(scantokens_group(raw));
212            out.push(T_OTHER!("]"));
213          }
214        },
215        // `\CPT@read@m`: a `{...}` group is read raw and re-scanned
216        // (`\CPT@read@mbeg`); a single token is passed straight through
217        // (`\CPT@read@mone`) — there is nothing to protect in one token.
218        'm' => match read_non_space()? {
219          Some(token) if token.get_catcode() == Catcode::BEGIN => {
220            out.extend(scantokens_group(read_protected_group()?));
221          },
222          Some(token) => out.push(token),
223          None => break,
224        },
225        // cprotect also knows `d`, a user-chosen delimiter pair whose two
226        // delimiters `\CPT@read@d` picks up from the input; an unknown letter is
227        // a package error there. We are more forgiving in both cases: an
228        // unhandled signature letter consumes nothing, so the command still
229        // reaches the digester and reads that argument itself — unprotected,
230        // but no worse than without `\cprotect`.
231        _ => {},
232      }
233    }
234    Ok(Tokens::new(out))
235  });
236
237  // `\newcommand{\cMakeRobust}[1]{...}` (cprotect.sty): stash the old meaning
238  // under `\CPT@old@<name>` and redefine `<name>` to cprotect itself.
239  DefPrimitive!("\\cMakeRobust DefToken", sub[(target)] {
240    let name = target.to_string();
241    let saved = T_CS!(&format!("\\CPT@old@{}", name.trim_start_matches('\\')));
242    let_i(&saved, &target, None);
243    // The interesting targets are precisely the ones the engine LOCKS —
244    // `\section`/`\subsection`/… are `DefMacro!(…, locked => true)` so a
245    // document's own `\renewcommand\section` cannot destroy the semantic
246    // structure. `\cMakeRobust` is not that kind of redefinition: it keeps the
247    // original meaning (under `\CPT@old@…`) and only wraps it, so bypassing the
248    // lock here is the same "an addition, never a replacement of the body's
249    // intent" case that `\g@addto@macro` unlocks for (Perl `Package.pm` L2527,
250    // `local $UNLOCKED = 1`). The `:locked` flag itself is untouched, so a later
251    // `\renewcommand\section` from the source is still ignored.
252    let _unlock = local_state_unlocked_guard(true);
253    def_macro(target, None, Tokens!(T_CS!("\\cprotect"), saved), None)?;
254    Ok(())
255  });
256
257  // `\def\cprotEnv\begin{\CPTbegin}` + `\CPTbegin{#1}` re-emitting
258  // `\begin{#1} \protect\input <file> \end{#1}`. In LaTeXML an environment body
259  // is digested straight from the mouth — `\verb` inside `\begin{center}…` is
260  // already read with live catcodes — so the write-and-read-back round trip is
261  // a no-op. Expanding `\cprotEnv` to nothing leaves the following
262  // `\begin{env}` in place, which is that same net effect.
263  DefMacro!(T_CS!("\\cprotEnv"), None, Tokens!());
264  Let!("\\CPTbegin", "\\begin");
265});
