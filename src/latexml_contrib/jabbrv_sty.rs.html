1//! Stub for jabbrv.sty (journal-abbreviation package).
2//!
3//! jabbrv provides `\JournalTitle{<full name>}` which abbreviates journal names via
4//! big `jabbrv-ltwa-*.ldf` lists. It is shipped with papers (not in every TeX Live)
5//! and pulled in by classes like wlscirep via `\RequirePackage{jabbrv}`. Two problems
6//! with the real package under LaTeXML:
7//!   (1) the OmniBus class-fallback dep-scan can't load a shipped-only raw `.sty`
8//!       (notex gate) → `\JournalTitle` undefined (~42 CONVERR_1 papers), and
9//!   (2) when jabbrv IS loaded via the class `\RequirePackage` path it leaves an
10//!       `\emph` group imbalance (~95 errors — SHARED with Perl, which raw-loads it).
11//! So the real raw-load is strictly WORSE than leaving it out (Perl = 95 errors;
12//! Rust currently = 1). This stub provides jabbrv's public API cleanly, WITHOUT the
13//! `.ldf` abbreviation machinery: `\JournalTitle` outputs the full journal name
14//! (faithful enough for XML — the ISO abbreviation is a print convention), and the
15//! `\Define*` declarations are no-ops. Result: `\JournalTitle` defined, 0 errors —
16//! surpasses Perl's broken raw-load. Witness 2112.00489 (wlscirep).
17use latexml_package::prelude::*;
18
19LoadDefinitions!({
20  DefMacro!("\\JournalTitle{}", "#1");
21  DefMacro!("\\DefineJournalAbbreviation{}{}", "");
22  DefMacro!("\\DefineJournalException{}{}", "");
23  DefMacro!("\\DefineJournalPartialAbbreviation{}{}", "");
24  DefMacro!("\\DefineSpuriousJournalWord{}", "");
25});
