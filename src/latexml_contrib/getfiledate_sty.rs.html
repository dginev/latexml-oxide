1//! Stub for getfiledate.sty — prints a file's last-modification date
2//! (a boxed/inline date string). Output-irrelevant for our purposes.
3//!
4//! Why a stub: getfiledate.sty L20 `\RequirePackage{ltxnew}` and uses
5//! its `\new` allocator prefix at load time —
6//! `\new\dimen\gfd@width@tmp\gfd@width@tmp=\z@` (L29). ltxnew builds
7//! `\new` via a `\futurelet`-based prefix scanner (a long `\ifx\x\dimen
8//! → \newdimen` chain) that our raw-load doesn't execute faithfully, so
9//! `\new\dimen\gfd@width@tmp` never allocates the register and the
10//! immediately-following `\gfd@width@tmp=\z@` raises
11//! `Error:undefined:\gfd@width@tmp` at package-load time.
12//!
13//! Perl LaTeXML matches the EFFECTIVE skip: it reports getfiledate as a
14//! missing-file (`Can't find binding for package getfiledate`) and only
15//! deps-scans it, never executing the body — so the paper converts
16//! cleanly with `\getfiledate` simply absent. We mirror that by stubbing
17//! the public `\getfiledate` as a no-op (the 4 witness papers
18//! 1503.08338/1503.08341/1709.04899/1803.07118 all `\usepackage` it but
19//! never call it). A future paper that DOES call `\getfiledate` loses
20//! only the decorative file-date string — same as Perl today.
21use latexml_package::prelude::*;
22
23LoadDefinitions!({
24  // getfiledate.sty L22 does `\RequirePackage[table]{xcolor}` (guarded by
25  // \@ifpackageloaded). Perl's deps-scan loads xcolor too, so a paper
26  // relying on getfiledate to transitively provide \textcolor/\color
27  // still works. Mirror that — without it, stubbing getfiledate drops
28  // xcolor and `\textcolor` goes undefined (witness 1803.07118).
29  RequirePackage!("xcolor", options => vec!["table".to_string()]);
30  // \getfiledate[<keyvals>]{<file>} — newcommand\getfiledate[2][] in the
31  // real package (one optional arg, one required). Consume and emit
32  // nothing, matching Perl's effective skip.
33  def_macro_noop("\\getfiledate[]{}")?;
34});
