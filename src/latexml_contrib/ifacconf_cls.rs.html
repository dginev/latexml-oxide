1//! ifacconf.cls (IFAC conference proceedings) — intentionally has **no
2//! binding**: we raw-load the real `ifacconf.cls` exactly like Perl.
3//!
4//! Perl LaTeXML ships no `ifacconf.cls.ltxml`. When a paper supplies its own
5//! `ifacconf.cls` (the class is not in TeX Live, so papers must), Perl raw-loads
6//! it under the OmniBus skeleton and scans its real dependencies
7//! (`theorem, newlfont, natbib` — notably **not** hyperref). The class then
8//! defines its own frontmatter (`\author`, `\address`, `\sep`, `\thanks`, …).
9//! We now mirror that: this module is unregistered, so a real `ifacconf.cls`
10//! falls through `find_class` → raw-load (OmniBus skeleton), matching Perl
11//! byte-for-structure. Task #273 (shrink OmniBus stubs via raw interpretation).
12//!
13//! This replaces an earlier Elsevier-style stub (`LoadClass("OmniBus")` plus
14//! eager `RequirePackage("amsmath"/"amssymb"/"amsthm"/"hyperref"/"graphicx"/
15//! "natbib")` and hand-rolled `\sep`/`\ead`/`\fnref`/… helpers). The stub's
16//! eager `\RequirePackage{hyperref}` was a Perl divergence: it bound `\url` to
17//! hyperref's verbatim reader, so a `.bbl`'s `\providecommand{\url}[1]{...}`
18//! (and a nested `\url{\url{...}}`) no longer collapsed to `\texttt`, producing
19//! a group/mode-frame cascade (38 errors vs Perl 0). Witness arXiv:1611.06249.
20//! The real class's own frontmatter is a strictly more faithful match than the
21//! Elsevier guess, and raw-load converts 1611.06249 with **zero** errors.
22//!
23//! NOTE (eager-xcolor cluster — referenced from the other journal `*_cls.rs`):
24//! do **not** eagerly `RequirePackage("xcolor")` from a class binding when Perl
25//! ships none. Preloading xcolor means a later document-level
26//! `\usepackage[table]{xcolor}` (or any `\usepackage[<opts>]{xcolor}`) sees
27//! xcolor as already-loaded and silently drops its options — so the `table`
28//! option never fires `\RequirePackage{colortbl}`, `array` is never pulled, and
29//! `m{<frac>\textwidth}` / `b{}` column types become "Unrecognized tabular
30//! template" → cascading "Extra alignment tab" errors. The document loads
31//! xcolor itself when it needs it (with the right options). Witness
32//! arXiv:2004.03970 (`\usepackage[table]{xcolor}` + `m{0.10\textwidth}` table →
33//! 8 errors in Rust, 0 in Perl). The same lesson applies to any eager
34//! `RequirePackage` of a package Perl leaves to the document.
