1//! aligned-overset.sty — visually re-centers `\overset`/`\underset` on the
2//! alignment point of an amsmath cell (so the *base* lines up on the `&`, not
3//! the accent). Purely a PDF-visual cosmetic with no MathML meaning: LaTeXML
4//! renders `\overset`/`\underset` as `<mover>`/`<munder>`, which centre
5//! intrinsically, so the `\hbox_set:`/`\box_wd:`/`\hspace` box-nudging the
6//! package performs is irrelevant to the output.
7//!
8//! The raw package is an expl3 (`\ProvidesExplPackage`) file that rewrites
9//! `\overset` to `\group_align_safe_begin: … \group_align_safe_end:` around an
10//! `\hbox_set:` measurement. That alignment-tab group trickery is not something
11//! LaTeXML's `\lx@begin@alignment` digestion can follow — when raw-loaded (only
12//! under INCLUDE_STYLES / the ar5iv profile; bare it is simply ignored) an
13//! `\overset` inside an `align` cell fires `\lx@begin@alignment Attempt to close
14//! a group that switched to mode math`, corrupts math mode for the rest of the
15//! block, and cascades into hundreds of `unexpected:_`/`^` once `_` is (correctly)
16//! catcode SUB. Binding it to keep amsmath's plain `\overset`/`\underset` and drop
17//! the cosmetic restores the output.
18//!
19//! Perl LaTeXML ships no `aligned-overset` binding either — it raw-loads and hangs
20//! (witness 2203.05327: same-host Perl 6m19s → `Fatal:timeout:token_limit`, 0 bytes),
21//! so this is a beyond-Perl faithfulness win. Witness 2203.05327 (ar5iv): 411
22//! errors → 0, 831 KB → full paper.
23use latexml_package::prelude::*;
24
25LoadDefinitions!({
26  // aligned-overset does `\RequirePackage{xparse,amsmath,mathtools}`. Provide the
27  // user-facing math deps (so `\overset`/`\underset` and mathtools commands exist
28  // even if the document loads only aligned-overset), but drop xparse — it was
29  // pulled in solely for the `\NewExpandableDocumentCommand` that implements the
30  // visual cosmetic we intentionally omit.
31  RequirePackage!("amsmath");
32  RequirePackage!("mathtools");
33});
