1//! Stub for lmcs.cls (Logical Methods in Computer Science journal class).
2//!
3//! lmcs.cls is NOT distributed in TeX Live (papers bundle it, but the corpus
4//! copies don't reach our search path), so Perl LaTeXML — which ships no lmcs
5//! binding — emits `Can't find binding for class lmcs (using OmniBus)` and
6//! falls back to **OmniBus**. We mirror Perl by loading OmniBus as the base
7//! (NOT amsart, which the prior version used): OmniBus supplies the lazy
8//! theorem-env autoloads (`\begin{thm}`/`\begin{lem}`/… → `\newtheorem{thm}`
9//! /`{lem}`/…), so a paper whose preamble does `\newtheorem{remark}[thm]{…}`
10//! and body uses `\begin{thm}`/`\begin{lem}` resolves the shared `thm` counter
11//! exactly as Perl does. amsart pre-defines none of those, so Rust hit
12//! `undefined:{thm}`/`{lem}`/`\thethm`. Witness 1607.01886 (RUST 3 → 0; 12
13//! theorems matching Perl). The raw cls relies on pgfmath/tikz/XeTeXLinkBox for
14//! ORCID-logo rendering (fails mid-load), so `\lmcsdoi`/`\lmcsheading`/
15//! `\lmcsorcid` get content-preserving stubs below. Witness 2305.14448,
16//! 2305.19985, 1607.04128 (graphicx).
17use latexml_package::prelude::*;
18
19LoadDefinitions!({
20  LoadClass!("OmniBus");
21  RequirePackage!("amsmath");
22  RequirePackage!("amssymb");
23  RequirePackage!("amsthm");
24  // lmcs.cls L33 `\LoadClass[11pt,reqno]{amsart}`. Perl ships no lmcs binding,
25  // so it loads OmniBus AND dependency-scans the raw lmcs.cls, which loads
26  // amsart.cls.ltxml → ams_support.sty.ltxml. That is where the amsart-family
27  // frontmatter macros (`\urladdr`, `\address`, `\email`, `\curraddr`) come
28  // from. Without it, `\urladdr{\url{…}}` was undefined (witness 1709.06170,
29  // RUST 1 → 0; Perl loads it via the amsart dep). ams_support is frontmatter
30  // only — it does not pre-declare theorem envs, so the OmniBus lazy
31  // `\begin{thm}` autoloads that 1607.01886 relies on are untouched.
32  RequirePackage!("ams_support");
33  RequirePackage!("hyperref");
34  // lmcs.cls L24 `\usepackage{helvet,cclicenses}` pulls graphicx in
35  // transitively: cclicenses → `\RequirePackage{rotating}` (cclicenses.sty
36  // L25) → `\RequirePackage{graphicx}` (rotating.sty L62). Perl ships no lmcs
37  // binding, so it raw-loads lmcs.cls and gets graphicx that way; LMCS papers
38  // therefore use `\includegraphics` WITHOUT their own `\usepackage{graphicx}`.
39  // Our stub intercepts the raw cls (which fails mid-load on its tikz/
40  // XeTeXLinkBox ORCID machinery), so the transitive graphicx never loaded
41  // and `\includegraphics` was undefined where Perl is clean. Supply it
42  // directly. Witness 1607.04128 (`\documentclass{lmcs}`, `\includegraphics`
43  // with no explicit graphicx load): RUST 1 → 0.
44  RequirePackage!("graphicx");
45  // Eager xcolor preload removed for Perl parity: it makes a later document
46  // xcolor[table] load a no-op, so colortbl/array never load and array m{}/b{}
47  // columns break (Unrecognized tabular template -> Extra alignment tab). The
48  // document loads xcolor itself; color/definecolor stay via hyperref->color.
49  // See ifacconf_cls.rs and SYNC_STATUS (eager-xcolor cluster).
50  RequirePackage!("enumitem");
51  RequirePackage!("etoolbox");
52
53  // LMCS publication metadata. Real macros assign internal counters
54  // and set up running headers; for HTML rendering we just preserve
55  // the args as named notes.
56  DefMacro!(
57    "\\lmcsdoi{}{}{}",
58    "\\@add@frontmatter{ltx:note}[role=lmcs-doi]{Volume #1, Issue #2, Paper #3}"
59  );
60  // \lmcsheading{vol}{issue}{year}{pages}{subm}{publ}{rev}{spec_iss}{title}
61  // The raw cls signature is 7-args but with optional/positional variations.
62  // We don't reproduce the running-header layout — just discard, since the
63  // metadata is already captured by \lmcsdoi.
64  def_macro_noop("\\lmcsheading{}{}{}{}")?;
65  // \lmcsorcid{orcid-id} — render as a plain link rather than the
66  // tikz/XeTeXLinkBox logo construction.
67  DefMacro!("\\lmcsorcid{}", "\\href{https://orcid.org/#1}{ORCID:#1}");
68
69  // Section-numbering and shortauthors/shorttitle helpers used by raw
70  // cls header layout. Stub as no-op or pass-through.
71  def_macro_noop("\\shorttitle{}")?;
72  def_macro_noop("\\shortauthors{}")?;
73
74  // `\dOi` placeholder produced by the raw cls when no \lmcsdoi was
75  // declared. Stub to empty so it doesn't appear as red error text.
76  def_macro_noop("\\dOi")?;
77});
