1//! aistats2026.sty — AISTATS 2026 conference style file.
2//!
3//! Identifies as a class-style file (`\documentclass{article}` +
4//! `\usepackage{aistats2026}`). The raw .sty includes a measurement-
5//! based `\PackageError{Document}{Running heading author exceeds size
6//! limitations}` that fires when the rendered running head exceeds
7//! page width (aistats2026.sty L289-295). Since we render to XML/HTML
8//! rather than PDF, page-width measurement is irrelevant and this
9//! error is moot in our paradigm. Pre-define `\runningauthor` to a
10//! benign value and stub `\@runningauthor` after raw load so the
11//! measurement loop short-circuits without firing PackageError.
12//!
13//! Witness: 47 papers across stages 15-20 fail with this exact error.
14use latexml_package::prelude::*;
15
16LoadDefinitions!({
17  // Pre-set \runningauthor to a placeholder so aistats2026.sty's
18  // `\ifx\undefined\@runningauthor` test sees it as already defined
19  // and skips the auto-fill from \@author (which is what triggers
20  // the over-width PackageError).
21  RawTeX!(r"\gdef\runningauthor#1{\gdef\@runningauthor{#1}}");
22  RawTeX!(r"\gdef\@runningauthor{}");
23
24  // Load the actual style for everything else (frontmatter, fancyhead
25  // setup, etc.). The runningauthor pre-init above prevents the
26  // \PackageError firing.
27  InputDefinitions!("aistats2026", noltxml => true,
28    extension => Some(Cow::Borrowed("sty")));
29});
