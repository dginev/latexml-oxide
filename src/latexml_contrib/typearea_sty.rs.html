1//! No-op stub for typearea.sty (KOMA-Script page layout).
2//!
3//! typearea defines KOMA's typearea page-layout system; the user-facing
4//! option is `DIV=<n>` (computed type-area divisions) and its variants.
5//! All KOMA-Script page-layout settings are typesetting-only — XML/HTML
6//! output never observes them — but our raw-load of typearea triggers
7//! `\RequirePackage{scrkbase}` → `scrbase`, and scrbase's option
8//! processing emits `Error:latex:\GenericError Package scrbase Error:
9//! unknown option` when it can't parse `DIV=11` (the keyval setup
10//! doesn't survive raw-load fully).
11//!
12//! Perl LaTeXML never observes this: its default TEXINPUTS excludes
13//! `/usr/share/texlive`, so typearea.sty is reported as missing-file
14//! and skipped, completing the conversion cleanly. Match that by
15//! registering a no-op binding for typearea.sty.
16//!
17//! Witness clusters (CONVERR_5 .. CONVERR_8 with scrbase noise):
18//! 1504.00554, 1502.06768, 1504.00666, and the ~6 R-stage papers in
19//! the `Package scrbase Error: unknown option` cluster.
20
21use latexml_package::prelude::*;
22
23LoadDefinitions!({
24  // `\areaset[BCOR]{width}{height}` — typearea's explicit page-area
25  // setter. Pure typesetting, no semantic effect for XML output.
26  // Witness: 1502.06768 uses `\areaset{...}{...}` to manually pin the
27  // typearea instead of using `DIV=...`.
28  def_macro_noop("\\areaset[]{}{}")?;
29});
