1//! tabls.sty — table layout tweaks (variable line height/depth).
2//!
3//! tabls redefines `\@array` to wrap its expansion with a
4//! `\@unrecurse` cleanup that restores global table-layout
5//! parameters. The redefinition lives inside `\@array`'s body, so
6//! `\@unrecurse` only gets `\edef`'d at `\@array` invocation. Our
7//! engine's tabular machinery bypasses the kernel `\@array` path,
8//! so `\@unrecurse` is never defined — when the (also-redefined)
9//! `\@xtabularcr` later calls `\@unrecurse`, it fires undefined.
10//!
11//! Witness arXiv:2003.12942 — `\usepackage{tabls}` + tabular env
12//! emits `\@unrecurse undefined` once per table row. Perl LaTeXML
13//! has no tabls binding and INCLUDE_STYLES=false skips raw load.
14//!
15//! Stub: pre-define `\@unrecurse` as `\relax` so the cleanup is
16//! a no-op. The visual fidelity loss is minor: our tabular
17//! rendering doesn't honour tabls's per-cell strut adjustments
18//! anyway.
19use latexml_package::prelude::*;
20
21LoadDefinitions!({
22  Warn!(
23    "missing_file",
24    "tabls.sty",
25    "tabls.sty is minimally stubbed — line-height tweaks are not honored; \\@unrecurse pre-defined as \\relax."
26  );
27  // Pre-define \@unrecurse before raw-load (defensive: in case
28  // our array path never instantiates the inner \edef).
29  Let!("\\@unrecurse", "\\relax");
30  // tabls's tabular customizations don't translate to HTML layout;
31  // leave the kernel tabular machinery intact. Don't raw-load.
32});
