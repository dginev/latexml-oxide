1//! mhchem.sty — chemical formula typesetting (`\ce`, `\cee`, `\cf`, `\cesplit`,
2//! `\bond`, …).
3//!
4//! **2026-06-27: RAW-LOAD the genuine TL `mhchem.sty`.** Perl LaTeXML ships no
5//! `mhchem.sty.ltxml` and raw-loads the real package; the engine's expl3 /
6//! xparse / chemgreek support is now mature enough that we can do the same, so
7//! this binding simply `InputDefinitions(noltxml)` the real file. Chemistry
8//! therefore renders with proper digit subscripts (`\ce{H2O}` → H₂O), charge
9//! superscripts (`\ce{SO4^2-}` → SO₄²⁻), reaction arrows (`->`/`<=>`/`->[..]`),
10//! bonds, states, `\cesplit`, etc. — full fidelity, matching Perl.
11//!
12//! This SUPERSEDES the former minimal "`\ce`-as-roman-text" stub (retired here;
13//! see git history at `latexml_contrib/src/mhchem_sty.rs` before this commit).
14//! The stub hand-shimmed a handful of behaviours (digit→subscript was NOT among
15//! them — formulae rendered flat), plus `amsmath`/`graphicx` `RequirePackage`s
16//! to paper over the auto-dep-scan gap; the real `mhchem.sty` runs its own
17//! `\RequirePackage{expl3,amsmath,calc,…}` chain, so none of that is needed.
18//!
19//! Known edge-case residuals under raw-load (tracked in docs/SYNC_STATUS.md):
20//! `\ce` used inside an amsmath `align*` alignment, and a few `$`-toggle /
21//! `\cesplit`-derived example patterns, can still emit `\lx@begin@alignment` /
22//! `\lx@end@inline@math` errors. Simple `\ce` (the overwhelmingly common case)
23//! is clean.
24use latexml_package::prelude::*;
25
26#[rustfmt::skip]
27LoadDefinitions!({
28  InputDefinitions!("mhchem", noltxml => true, extension => Some(Cow::Borrowed("sty")));
29});
