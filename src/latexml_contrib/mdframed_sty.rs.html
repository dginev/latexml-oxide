14  // `\newmdenv[opts]{name}` defines a new environment `name` that wraps
15  // `mdframed` (mdframed.sty L578-585:
16  //   \newenvironment{#2}{\mdfsetup{#1}\begin{mdframed}}{\end{mdframed}}).
17  // `\mdfsetup` is our no-op, so the body reduces to a mdframed wrapper.
18  // Surpass-Perl: ar5iv-bindings/mdframed.sty.ltxml L22 also no-ops this,
19  // leaving the user's custom env undefined (Perl then errors with
20  // `{name} is not defined`). Faithfully porting the real definer makes
21  // the custom env work. Witness arXiv:2002.06879
22  // (`\newmdenv[...]{mdfigure}` then `\begin{mdfigure}`).

30  );

37  // Wrap body in `inline-logical-block` (Misc.class container with Para.model body).
38  //
39  // The schema offers three framed-box elements, each satisfying only TWO of
40  // the three placements an `mdframed` must support (verified against
41  // resources/RelaxNG: float_model ⊇ Block.model = Block.class|Misc.class|
42  // Meta.class; Para.model = Para.class|Meta.class):
43  //   * `inline-block`        (Misc.class, body=Block.model): in-float ✓, nests ✓ (Block.model ⊇
44  //     Misc.class), theorem ✗ (Block.model ⊉ Para.class). This is what Perl
45  //     ar5iv-bindings/mdframed.sty.ltxml L31-34 uses, so Perl ITSELF errors
46  //     `malformed:ltx:theorem` on a theorem-in-mdframed.
47  //   * `inline-logical-block`(Misc.class, body=Para.model): in-float ✓ (Misc.class ⊂ Block.model ⊂
48  //     float_model), theorem ✓ (Para.model ⊇ Para.class), nests ✗ — a directly-nested inner
49  //     `inline-logical-block` (Misc.class) isn't in the outer's Para.model.
50  //   * `logical-block`       (Para.class, body=Para.model): theorem ✓, nests ✓ (Para.class ∈
51  //     Para.model), in-float ✗ — Para.class ⊄ float_model.
52  //
53  // No single element does all three, and the missing auto-open bridge
54  // (inline-logical-block → para → inline-logical-block, which para_model =
55  // Block.model WOULD admit) is intentionally suppressed by the `($tag ne $kid)`
56  // self-nesting guard in BOTH Perl `Document::computeIndirectModel` (L207) and
57  // our `state::compute_indirect_model` — so adding it would diverge from Perl's
58  // document model. We therefore pick the element that fails the RAREST case:
59  //
60  // History: this was `logical-block` (theorem ✓ + nests ✓) until a fresh sweep
61  // surfaced arXiv:1907.05772 — an `mdframed` inside a `\begin{algorithm}`
62  // float, where Perl is clean (0 err, its `inline-block` is Misc.class) but
63  // `logical-block` (Para.class) tripped `"logical-block" isn't allowed in
64  // <float>` ×3 (Rust-only, Perl=0). mdframed-in-float (framed algorithm/figure
65  // boxes) is far more common than nested frames, so `inline-logical-block`
66  // strictly dominates `logical-block`: it FIXES the float regression and keeps
67  // the theorem-in-mdframed surpass (arXiv:2506.03074, 2402.07712 — beyond Perl,
68  // which errors there). The residual cost is the rare directly-nested-frame
69  // case (1712.00062): inner frame as the FIRST child of an outer frame errors
70  // `"inline-logical-block" isn't allowed in <inline-logical-block>` (any leading
71  // text auto-opens a `para` that then admits the inner frame, so only the
72  // bare-first-child variant is affected). Net: trades a moderate Rust-only
73  // regression (float) for a rare one (bare-nested-frame), maximizing error-free
74  // conversions.
75  //
76  // The template emits `framecolor=` only when the #framecolor property is
77  // set (via the `?#framecolor(...)` guard), so an unset color correctly
78  // omits the attribute rather than emitting `framecolor=''`.
79  DefEnvironment!(
80    "{mdframed}[]",
81    "<ltx:inline-logical-block framed='rectangle' ?#framecolor(framecolor='#framecolor') _noautoclose='1'>#body</ltx:inline-logical-block>",
82    properties => sub[_args] {
83      let mut props = SymHashMap::default();
84      if let Some(font) = lookup_font()
85        && let Some(color) = font.get_color() {
86          props.insert("framecolor", Stored::from(color.to_attribute()));
87        }
88      Ok(props)
89    },
90    // mdframed bodies routinely contain multi-paragraph content
91    // (theorems, displayed equations, multiple `$$..$$` blocks). The
92    // DefEnvironment default of restricted_horizontal makes
93    // BOUND_MODE never end with "vertical", so tex_math.rs:467's
94    // `$$` → display-math check stays false: each `$$` is parsed as
95    // open + immediate close, leaving body content in text mode and
96    // cascading "Script _/^ can only appear in math mode" on subscripts.
97    // Witness 2402.07712 (eqnarray + multiple `$$..$$` in mdframed).
98    mode => "internal_vertical");
99});
