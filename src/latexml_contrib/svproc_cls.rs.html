1//! svproc.cls (Springer Proceedings template, sister of svjour/svmult).
2//!
3//! Base = `article` + `sv_support` (Springer support), mirroring Perl's
4//! svjour.cls.ltxml / svmult.cls.ltxml which both
5//! `RequirePackage('sv_support')`. sv_support pulls inst_support + natbib
6//! and provides the Springer frontmatter (`\email`/`\keywords`/`\institute`)
7//! plus `\spnewtheorem` and the standard theorem set — so we do NOT need
8//! OmniBus (its last-resort grab-bag) nor an eager amsthm preload. See
9//! WISDOM #55. The paper's own `\usepackage{amsthm}` (if any) is then the
10//! first real amsthm load, so the `\let\proof\relax` idiom works (witness
11//! 1707.03222 {proof}).
12use latexml_package::prelude::*;
13
14LoadDefinitions!({
15  LoadClass!("article");
16  // svproc.cls has no class binding in Perl, so Perl falls back to OmniBus,
17  // whose `RequirePackage('graphicx')` (OmniBus.cls.ltxml L44 / our
18  // omnibus_cls.rs L46 — "loaded by random cls files 10-to-1") defines
19  // `\includegraphics`. This contrib binding intercepts svproc BEFORE the
20  // OmniBus fallback, so without this the class's figures hit
21  // `undefined:\includegraphics` and cascade (`_` in text mode). Mirror the
22  // OmniBus outcome. Witness 2004.07524 (svproc proceedings with `\includegraphics`).
23  RequirePackage!("graphicx");
24  RequirePackage!("sv_support");
25  RequirePackage!("amsmath");
26  // Pre-load xcolor with [dvipsnames, table] so a paper's later
27  // `\usepackage[table]{xcolor}` doesn't silently option-clash and
28  // leave colortbl unloaded → `\cellcolor` undefined. svproc.cls
29  // itself does NOT load xcolor (only uses the kernel `\normalcolor`),
30  // so in Perl the user's `[table]{xcolor}` is the first load and
31  // colortbl comes in via the `table` option; matching that outcome
32  // here. Same anti-clash pattern as mnras_cls / quantumarticle_cls.
33  // Witness: 1706.04315, 1804.09301, 1807.04749 ("undefined:\\cellcolor").
34  RequirePackage!("xcolor", options => vec!["dvipsnames".to_string(), "table".to_string()]);
35  RequirePackage!("hyperref");
36  // svproc.cls L1387-1466 inlines the full standard AAS journal-abbreviation
37  // set (`\newcommand*\apj{ApJ}`, `\aj`, `\mnras`, `\sovast`, … — 79 macros,
38  // the same set as aas_macros.sty). Bibliographies generated by AAS/natbib
39  // bst styles emit these directly in `\bibitem` text. Our binding intercepts
40  // svproc.cls, so those inline defs never run; mirror them via aas_macros
41  // (which Perl-faithfully wraps each as `\ref@jnl{…}`). Without this,
42  // `\apj`/`\sovast` are undefined where Perl (which raw-loads the bundled
43  // svproc.cls) is clean. Witness 2110.04152 (`\apj`, `\sovast` in thebibliography).
44  RequirePackage!("aas_macros");
45
46  // svproc.cls L864: \newtoks\tocauthor / \toctitle for TOC entries.
47  // Preserve author content as ltx:note.
48  DefMacro!(
49    "\\tocauthor{}",
50    "\\@add@frontmatter{ltx:note}[role=tocauthor]{#1}"
51  );
52  DefMacro!(
53    "\\toctitle{}",
54    "\\@add@frontmatter{ltx:note}[role=toctitle]{#1}"
55  );
56  DefMacro!(
57    "\\institute{}",
58    "\\@add@frontmatter{ltx:note}[role=institute]{#1}"
59  );
60  // \inst{N} is a superscript marker keyed to numbered affiliations.
61  DefMacro!("\\inst{}", "\\textsuperscript{#1}");
62  // svproc.cls L240-245 defines \frontmatter / \mainmatter / \backmatter as
63  // page-numbering + \@mainmatter-flag toggles (`\pagenumbering{Roman}` etc.).
64  // All three are pure page-layout — moot in our HTML paradigm — so Perl
65  // (which raw-loads svproc.cls and runs them) produces no visible output for
66  // them. Our binding intercepts svproc.cls, so the raw defs never run;
67  // `\mainmatter` was already stubbed but `\frontmatter`/`\backmatter` were
68  // left undefined where Perl is clean. No-op all three (matching their net
69  // HTML effect). Witness 1902.03320 (`\documentclass{svproc}`, `\frontmatter`).
70  def_macro_noop("\\frontmatter")?;
71  def_macro_noop("\\mainmatter")?;
72  def_macro_noop("\\backmatter")?;
73});
