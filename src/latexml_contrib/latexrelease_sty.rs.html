1//! Stub for latexrelease.sty (LaTeX kernel rollback machinery).
2//!
3//! latexrelease.sty is a 17k-line LaTeX2e rollback package: papers
4//! that need a specific historical kernel date
5//! (`\RequirePackage[2020-02-02]{latexrelease}`) use it to re-bind
6//! kernel macros to older variants. The raw sty defines
7//! `\IncludeInRelease`/`\EndIncludeInRelease` block-pairs gated by
8//! kernel-date comparisons; inside those blocks it does
9//! `\let \@expl@str@if@eq@@nnTF \@undefined` and similar undef-then-
10//! rebind dance. Our system does NOT track kernel dates, so it executes
11//! every `\IncludeInRelease` block linearly â€” leading to undefined-CS
12//! cascades during the rollback machinery itself.
13//!
14//! In our HTML conversion pipeline, kernel-date rollback is meaningless
15//! (it controls pagination/font/parser internals that don't affect
16//! semantic content). Stub the whole package as a no-op. Witness:
17//! 2305.09466, 2305.12592, 2305.14116, 2305.16020, 2305.17060 (all
18//! 5-error conversion_fatal from \@expl@str@if@eq@@nnTF cascade after
19//! latexrelease.sty raw-load).

23  // No-op: skip rollback machinery entirely. If a future paper actually
24  // depends on `\IncludeInRelease` / `\EndIncludeInRelease` macro-pairs
25  // in user code (rather than just inside latexrelease.sty's own bundle),
26  // we'd stub those here too.
