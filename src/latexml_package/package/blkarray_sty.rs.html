1//! blkarray — block-array matrices.
2//!
3//! The raw `blkarray.sty` builds each matrix cell as inline math inside a raw
4//! `\halign`/`\ialign`, and its `block` delimiter machinery, digested inside
5//! display math, drives BOTH LaTeXML engines into the `\halign`-in-math runaway
6//! (Rust OOMs at the 4.5 GB cap in ~12 s; same-host Perl hangs ~90 s → rc=124;
7//! `pdflatex` renders fine). Full analysis + 4-line reproducer:
8//! `docs/known_crashes/blkarray_halign_math/`. This binding SHADOWS the raw
9//! `.sty` (so it is never raw-loaded, even under `--includestyles`) and routes
10//! `blockarray`/`block` through the engine's well-behaved `array` alignment
11//! machinery instead. Surpass-Perl: upstream LaTeXML has no `blkarray.sty.ltxml`.
12//! Witnesses: arXiv:1811.10792 (ar5iv #594), arXiv:2310.17416 (ar5iv #473).
13//!
14//! **Faithfulness note (documented simplification).** In `blkarray`, a `block`'s
15//! column-spec delimiters (`(`/`[` in e.g. `block{c(cccccc)}`) wrap a SUB-REGION
16//! of the shared matrix — a construct LaTeXML's `array` cannot express (its
17//! `left=`/`right=` wrap the whole matrix). We render each `block` transparently:
18//! its rows flow into the single `blockarray` alignment (correct structure +
19//! content, and the label row/column are preserved), but the block's delimiter
20//! parentheses are DROPPED. This is chosen deliberately over the raw `.sty`'s OOM
21//! — a matrix without its outer parens beats losing the whole document section.
22use crate::prelude::*;
23
24LoadDefinitions!({
25  // `\begin{blockarray}[pos]{spec}` is defined as a MAGIC control sequence
26  // (`\begin{...}` takes the defined-CS fast path in latex_constructs.rs:3064,
27  // which does NOT inject a `\begingroup`). That is essential: a transparent
28  // `block` nested inside must not open a group that crosses the alignment's
29  // `\\` row boundaries. Route to the same machinery as `\array`
30  // (latex_constructs.rs `\@array@bindings`/`\@@array`/`\lx@begin@alignment`).
31  // blkarray's own blockarray column specs are plain (`c cccccc`, `cccc`) — the
32  // delimiters live on the block specs, which we gobble — so the spec passes to
33  // the AlignmentTemplate parser unchanged.
34  DefMacro!(
35    T_CS!("\\begin{blockarray}"),
36    "[]{}",
37    "\\@array@bindings[#1]{#2}\\@@array[#1]{#2}\\lx@begin@alignment"
38  );
39  DefMacro!(
40    T_CS!("\\end{blockarray}"),
41    None,
42    "\\lx@end@alignment\\@end@array"
43  );
44
45  // `block` / `block*` are transparent: gobble the column spec (which may carry
46  // `(`, `[`, `|` delimiters we cannot render sub-region-wise) and contribute
47  // their rows directly to the enclosing blockarray alignment. Magic CSes → no
48  // `\begingroup`, so the block boundary does not disturb the alignment grouping.
49  DefMacro!(T_CS!("\\begin{block}"), "{}", "");
50  DefMacro!(T_CS!("\\end{block}"), None, "");
51  DefMacro!(T_CS!("\\begin{block*}"), "{}", "");
52  DefMacro!(T_CS!("\\end{block*}"), None, "");
53});
