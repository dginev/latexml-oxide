11  // Real lineno.sty also defines control sequences `\linenomath`,
12  // `\linenomathWithnumbers`, `\linenomathNonumbers` (raw-load
13  // sees these as macros). Other packages — eccv.sty, journal templates —
14  // test them with `\ifx\linenomath\linenomathWithnumbers` to switch
15  // between AMS-math styles. Without explicit defs here, all three resolve
16  // to `\relax` and the `\ifx` test is TRUE — the then-branch fires
17  // `\patchcmd\linenomathAMS{...}` which is undefined → cascade of
18  // `\else` / `\fi` mismatch (27 of 44 wp4 \else-error papers use eccv).
19  // Make them three *distinct* no-op macros so the `\ifx` test picks the
20  // else-branch reliably, matching the no-linenumbers default.
21  // Don't redefine `\linenomath` / `\endlinenomath` — those are the
22  // env-begin/env-end macros set up by DefEnvironment above. We DO
23  // define the two "style switch" macros that real lineno provides,
24  // with distinct bodies so journal-template `\ifx\linenomath\linenomathWithnumbers`
25  // tests reliably pick the no-linenumbers branch.
26  DefMacro!("\\linenomathWithnumbers", "\\relax");
27  DefMacro!("\\linenomathNonumbers",   "\\@empty");
28  // \internallinenumbers (lineno.sty) — adds line numbers inside the

30  // `\@namedef{internallinenumbers*}{\internallinenumbers*}` at lineno
31  // L?? (cf. iclr2025_conference.sty L230 which calls it). Stub as
