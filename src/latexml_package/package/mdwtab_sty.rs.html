1//! mdwtab.sty — alternative tabular environment by Mark Wooding (1996).
2//!
3//! mdwtab.sty completely reimplements `\tabular` with its own
4//! `\tab@*` preamble-parsing machinery (`\tab@right`,
5//! `\tab@restorehlstate`, `\tab@bgroup`, `\tab@multicol`, etc.).
6//! Raw-loading it would redefine our locked `\tabular` binding —
7//! the redefinition is silently ignored, but then mdwtab's own
8//! `\tab@*` helpers (referenced by every column type and `\@arstrut`
9//! handler) end up undefined when the user actually enters a
10//! tabular, cascading through `\omit`/`\@startsection` mode errors
11//! to a TooManyErrors fatal.
12//!
13//! Perl LaTeXML has no `mdwtab.sty.ltxml`; with its default
14//! `INCLUDE_STYLES=false` the raw `mdwtab.sty` is NOT loaded —
15//! Perl emits a "missing binding" warning and continues with the
16//! binding-aware `\tabular`. The user's tables then render
17//! through standard array/tabular machinery (no mdwtab-specific
18//! ornaments, but cleanly).
19//!
20//! In Rust we default to `INCLUDE_STYLES=true` (ar5iv preload sets
21//! it). This stub suppresses the raw-load so `\tabular` keeps
22//! pointing at our binding-aware constructor. Same pattern as
23//! `delarray_sty.rs` / `trace_sty.rs`.
24//!
25//! Witness: canvas-3 stage-26 0910.3293 (uses `\usepackage{mathenv}`
26//! which `\RequirePackage{mdwtab}`).
