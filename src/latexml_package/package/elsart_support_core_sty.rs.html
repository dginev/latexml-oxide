154  // Perl elsart_support_core.sty.ltxml L138-141: `\keyword{...}` (1-arg
155  // form) reads a balanced group and wraps with `\@keyword <arg>
156  // \@keyword@cut` so XUntil terminates at the inserted token.
157  //
158  // History — DO NOT switch this back to `read_balanced(_, _, false)`:
159  // commit 09bc60c2 ("\keyword absorbs trailing unbalanced }") tried to
160  // mirror Perl's `$gullet->readBalanced` with no args (require_open=
161  // false → level starts at 1) to absorb the legacy unbalanced-`}`
162  // idiom from arXiv:1710.03688 / hep-ph0702114, where the abstract
163  // ends:
164  //     \keyword{Keyword1; ...} \vskip ...\noindent{...e3}}
165  // (a trailing literal `}` past the keyword arg). For that input
166  // alone, `require_open=false` correctly consumes `{...} }` and stops.
167  //
168  // BUT for the common balanced form — `\keyword{Higgs; Boson}` with
169  // no trailing extra `}` (the test fixture
170  // `tests/babel/elsart_keyword_brace_form.tex` and the vast majority
171  // of real-world usage) — `require_open=false` reads past the
172  // matching `}`, walks through `\end{abstract}` (raw `\end` is a CS
173  // and CC_END count is unchanged across that expansion), then
174  // `\end{frontmatter}`, … all the way to EOF. The reader's pushback
175  // grows unboundedly on the way; in CI we observed a single
176  // `memory allocation of 21743271936 bytes failed` (21 GB) → SIGABRT
177  // before the test could even report. The cost of one paper's
178  // 1-error reduction is the entire test suite OOM-aborting at
179  // `81_babel::elsart_keyword_brace_form_test`.
180  //
181  // RESOLVED 2026-05-30 via the design note's first suggestion: keep the
182  // strict balanced read of the keyword argument, then peek past spaces and
183  // gobble an OPTIONAL trailing `}` (catcode END). This reproduces Perl's
184  // lenient `readBalanced` *result* (it absorbs the legacy unbalanced `}`)
185  // for the trailing-`}` idiom — abstract bodies that end
186  //     \keyword{Kw1; Kw2; …}   <comments/spaces>   }   \end{abstract}
187  // (the stray `}` is the orphaned close of a commented-out brace group;
188  // witness 1601.01227, also 1710.03688 / hep-ph0702114) — WITHOUT the
189  // speculative read-to-EOF that OOM'd the balanced fixture: for the common
190  // `\keyword{Higgs; Boson}\end{abstract}` form the token after the strict
191  // read is `\end` (a CS, not `}`), so nothing is gobbled. The peek is
192  // bounded to one token, never crossing `\end`.
193  // Skip leading spaces and blank-line `\par`s, then gobble a single trailing
194  // `}` (catcode END) if one is there — Perl's `readBalanced` reads straight
195  // through the `\par` from a blank line before the stray brace (witness
196  // 1705.01354 separates `\keyword{…}` and the orphaned `}` by a blank line).
197  // The skipped whitespace/`\par` tokens are buffered and FULLY RESTORED when
198  // no trailing `}` follows, so the common `\keyword{…}\end{abstract}` form
199  // (and `\keyword{…}<blank>\end{abstract}`) is left byte-for-byte untouched.
200  // Bounded: stops at the first non-space, non-`\par` token; never crosses
201  // `\end`.
202  // Perl's `readBalanced` (L140) reads tokens up to the next UNMATCHED `}`
203  // (tracking brace depth), absorbing that stray `}` as its terminator —
204  // regardless of what non-brace material (spaces, `\par`, `\vskip <dimen>`,
205  // `\noindent`, …) sits between `\keyword{…}` and the orphaned brace. The
206  // earlier version only skipped spaces/`\par`s, so it stopped at the first
207  // real token (e.g. `\vskip`) and left the stray `}` to hit the abstract's
208  // mode-switch frame (`ltx:para`-free `}` → "close a group that switched to
209  // mode internal_vertical"; witness 1604.00855:
210  // `\keyword{…} \vskip 0.5\baselineskip}`).
211  //
212  // Walk forward tracking brace depth: an unmatched `}` (depth 0) is the
213  // stray terminator — DROP it, re-inject everything else (so the `\vskip`
214  // etc. still render, content-preserving). Bound the scan at `\end` (depth
215  // 0) — the env terminator — and a hard token cap, so the common
216  // `\keyword{Higgs; Boson}\end{abstract}` form (no stray `}`) reads one
217  // token (`\end`), restores it, and is left untouched (this is also what
218  // averts the read-to-EOF OOM a naive `readBalanced` hit). Witnesses:
219  // 1604.00855, 1601.01227, 1705.01354, 1710.03688, hep-ph0702114.
220  DefPrimitive!("\\lx@elsart@gobble@optbrace", {
221    let mut skipped: Vec<Token> = Vec::new();
222    let mut depth: i32 = 0;
223    let mut count: usize = 0;
224    while let Some(tok) = read_token()? {
225      count += 1;
226      if count > 4096 {
227        skipped.push(tok); // safety cap: restore everything, gobble nothing
228        break;
229      }
230      let cc = tok.get_catcode();
231      if cc == Catcode::BEGIN {
232        depth += 1;
233        skipped.push(tok);
234      } else if cc == Catcode::END {
235        if depth == 0 {
236          break; // unmatched stray `}` — drop it (absorbed, like readBalanced)
237        }
238        depth -= 1;
239        skipped.push(tok);
240      } else if depth == 0 && tok == T_CS!("\\end") {
241        skipped.push(tok); // env end reached before any stray `}` — restore all
242        break;
243      } else {
244        skipped.push(tok);
245      }
246    }
247    // Re-inject everything we buffered, in order. The only token ever removed
248    // is the stray `}` (which is never pushed onto `skipped`).
249    for tok in skipped.into_iter().rev() {
250      unread_one(tok);
251    }
252  });
