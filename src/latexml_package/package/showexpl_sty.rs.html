1//! No-op stub for showexpl.sty.
2//!
3//! showexpl.sty (TL) builds the `LTXexample` environment and
4//! `\LTXinputExample` command (display LaTeX source + its typeset
5//! result side by side) on top of `listings`. Its package body uses
6//! `\lst@newenvironment{LTXexample}{...}{...\SX@put@code@result}`
7//! whose end-group `\xdef\SX@@explpreset{\the\@temptokena,...}` parse
8//! tickles a readBalanced `Expected opening '{'` error in our raw-load
9//! path, then `\SX@put@code@result` (defined just after, at L208)
10//! never registers, cascading.
11//!
12//! Perl LaTeXML never observes this: its default TEXINPUTS excludes
13//! `/usr/share/texlive`, so showexpl.sty is reported as missing-file
14//! and skipped, completing cleanly. Verified on arXiv:2002.09910
15//! (`\usepackage{...showexpl...}`): Perl emits
16//! `Warning:missing_file:showexpl` and 0 errors.
17//!
18//! Match Perl's effective behavior with a stub that (a) pulls in
19//! showexpl's real `\RequirePackage` dependency chain so any
20//! transitively-needed macros (listings, varwidth, float) resolve, and
21//! (b) no-ops the user-facing showexpl commands. All 15 R-stage papers
22//! blocked on `\SX@put@code@result` were checked: NONE invoke
23//! `\begin{LTXexample}` or `\LTXinputExample` — they only load the
24//! package — so the stub costs no document content.
25//!
26//! Witnesses (CONVERR_7/CONVERR_3 → OK): 1604.00381, 1606.01035,
27//! 1706.03232, 1804.02704, 1804.07221, 1612.01022, 1905.12059,
28//! 1706.09226, 1701.01402, 1812.06820, 1801.01025, 1806.10927,
29//! 2001.08314, 2002.09910, 1901.08750.
30
31use crate::prelude::*;
32
33#[rustfmt::skip]
34LoadDefinitions!({
35  // showexpl.sty L1-13 \RequirePackage chain (attachfile is loaded
36  // conditionally via `\IfFileExists`; we load it unconditionally —
37  // harmless, has a binding).
38  RequirePackage!("listings");
39  RequirePackage!("refcount");
40  RequirePackage!("varwidth");
41  RequirePackage!("float");
42  // User-facing showexpl API — no-op (these display code+result side by
43  // side, which our XML output models as plain listings; no paper in
44  // the observed cluster uses them, so a no-op is safe and matches
45  // Perl's missing-file skip).
46  def_macro_noop("\\LTXinputExample[]{}{}")?;
47  def_macro_noop("\\setupSXfiles")?;
48  def_macro_noop("\\setupLZfiles")?;
49});
