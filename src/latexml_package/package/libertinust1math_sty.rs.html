1//! libertinust1math.sty — Libertine Unicode math font support.
2//!
3//! Same `\libus@undefine#1\let#1=\@undefined` pattern as fdsymbol.sty
4//! (see [`crate::package::fdsymbol_sty`] for the full rationale): the
5//! package re-declares every math symbol by first
6//! `\let\X\@undefined`-ing it and then `\DeclareMathSymbol{\X}{...}`-ing
7//! it back. Our kernel math chars (`\prime`, `\sum`, `\int`, ...) are
8//! locked at `DefMath!` time, so the re-DeclareMathSymbol is silently
9//! blocked and the previous `\let \@undefined` wins, cascading
10//! "undefined:\prime" errors for every formula that uses primes.
11//!
12//! Perl LaTeXML never raw-loads libertinust1math.sty (Perl-default
13//! `INCLUDE_STYLES=false`), so the kernel `\prime`/`\sum`/... stay
14//! intact. Match that behaviour: no-op binding. Font choice is moot for
15//! XML/HTML output. Witness 2406.04255, 2406.04389, 2407.16071
16//! (~28 papers cumulative, up to 501 `\prime` errors per paper).

18
19LoadDefinitions!({
20  // Intentionally empty — kernel math chars are authoritative.
21});
