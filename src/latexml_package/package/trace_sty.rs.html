1//! trace.sty — debug-only macro-tracing wrapper.
2//!
3//! trace.sty (F. Mittelbach, 1999-2019) wraps `\frozen@everymath`,
4//! `\frozen@everydisplay`, `\define@newfont`, `\calc@open`, and
5//! `\maybe@ic@` with `\conditionally@traceoff`/`\conditionally@traceon`
6//! pairs so that `\traceon`/`\traceoff` can selectively dump
7//! `\tracingmacros`-style output during compilation. The package
8//! has no semantic effect — its job is to write log lines.
9//!
10//! Raw-loading trace.sty trips our long-standing `\newtoks` slot-detach
11//! bug (see `latexml_core/src/dump_writer.rs` L176-211 for the dump-time
12//! workaround). trace.sty's reassignment of `\frozen@everymath = {...
13//! \the\everymath}` lands on `\everymath`'s slot, making subsequent
14//! math-mode entry expand `\the\everymath` recursively until the
15//! conditional limit (8M) is exhausted.
16//!
17//! Stub the package as a no-op: define the user-visible knobs
18//! (`\traceon`, `\traceoff`, `\tracingall`, `\conditionally@traceoff`,
19//! `\conditionally@traceon`) as empty macros so user-defined wrappers
20//! that call them don't error, then return. The package's diagnostic
21//! tracing output is irrelevant for our XML/HTML rendering paradigm.
22//!
23//! Witnesses: canvas-3 stage-23 0812.0208 (`\documentclass{ptptex}`
24//! → OmniBus + trace.sty raw-load → `\maketitle` triggers
25//! `\frozen@everymath` loop on the `$^1$` in `\author{... \textsc{Y}$^1$
26//! ...}`); minimal 7-line repro likewise fixed.
27use crate::prelude::*;
28
29LoadDefinitions!({
30  def_macro_noop("\\traceon")?;
31  def_macro_noop("\\traceoff")?;
32  def_macro_noop("\\tracingall")?;
33  def_macro_noop("\\conditionally@traceoff")?;
34  def_macro_noop("\\conditionally@traceon")?;
35  def_macro_noop("\\unconditionally@traceoff")?;
36  def_macro_noop("\\tr@ce@n")?;
37});
