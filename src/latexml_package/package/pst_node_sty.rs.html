14  // the untyped placeholder ("") since Rust pst-node stubs all connection
15  // commands to no-ops (DVI-only), so no consumer actually coerces these.

26  // Node definition macros — Perl L30-120. `\rnode` / `\pnode` etc. are
27  // Perl DefConstructor with coordinate-reading closures; Rust stubs them
28  // as DefMacro passthrough of the content (#3). DP-audit flags the kind
29  // flip. Structural — pst-node nodes need coordinate+position readers
30  // (same PSDim/PSAngle family as pstricks_support, WISDOM #41 parameter
31  // gap). Rust's passthrough preserves the label text in output; the
32  // geometric node-graph is simply not emitted. Not an `\edef` site.
33  //
34  // Intentional divergence (WISDOM #44 class: blocked-on-parameter-type):
35  // ALL node + connection stubs below share this single root cause —
36  // missing PSDim/PSAngle/PSDimDim/Float types — so the DefConstructor
37  // → DefMacro kind flips are a single-cluster waiver. Porting the
38  // PS* parameter-type family closes every rnode/Rnode/pnode/cnode/
39  // circlenode/ovalnode/trinode/dianode + nc*/pc* connection entry at
40  // once. Audit currently flags rnode+pnode explicitly.
41  DefMacro!("\\rnode[]{}{}", "#3");

44  // PSTricks node macros take a `(coord)` argument, not `[...]`. Perl
45  // pst-node.sty.ltxml: \cnode = 'ZeroPSCoord {PSDimension} {}'
46  // (coord, radius, name); \Cnode = 'ZeroPSCoord {}' (coord, name).
47  // The old `[]{}` signatures left the `(x,y)` coord unconsumed, so
48  // `\Cnode(1,1){000}` leaked "1,1)" as body text. Consume the optional
49  // `[par]`, the `(coord)`, then the braced args (mirrors the
50  // `\cnodeput ... [] () {} {}` pattern in pst_all_sty.rs).
