1//! `DocOwnedNode` — a drop-safe wrapper for libxml `Node` handles whose
2//! C storage is owned by the enclosing `Document`.
3//!
4//! # Why this exists
5//!
6//! The `libxml` crate's `_Node::drop` (v0.3.9 `src/tree/node.rs:71`)
7//! unconditionally calls `xmlFreeNode(ptr)` when the wrapper's internal
8//! `unlinked` flag is true. That flag is flipped by `Node::unlink_node()`
9//! and has no public setter — once a Node has been unlinked, letting
10//! Rust's drop glue run will always fire `xmlFreeNode`.
11//!
12//! For the unlinked *subtree* case we actually want the opposite: the
13//! node was detached from the live document topology but its memory
14//! (properties, namespace declarations, dict entries) is still shared
15//! with the enclosing `Document`. When `xmlFreeDoc` eventually walks
16//! the tree, it double-frees those shared allocations → SIGSEGV inside
17//! `xmlFreeNodeList` / `xmlFreeProp` (see `docs/known_crashes/README.md`
18//! and the 0705.0790 bisection).
19//!
20//! The prior fix for the idcache path was an in-place `std::mem::forget`
21//! call inside `PostDocument::drop` (`document.rs:84-113`). A second
22//! identical workaround landed for the `xmath` local in
23//! `process_math_node` (cycle 236 commit `18c9640ee`). Scattered
24//! `mem::forget`s are a code smell — they're easy to miss when a new
25//! caller also holds a lingering `Node` clone for an unlinked subtree.
26//!
27//! `DocOwnedNode` hoists the pattern into a single RAII type. A
28//! `DocOwnedNode` drops without invoking the inner `Node`'s Drop impl,
29//! so the `xmlFreeNode` call is skipped and the Document remains the
30//! sole owner of the C memory. The per-wrapper Rc control block
31//! (~24 B) leaks until process exit — bounded by how many subtrees
32//! the caller detaches.
33//!
34//! # Ergonomics note for upstream
35//!
36//! The proper fix is in `rust-libxml`: expose a public `relink()` or
37//! `mark_as_doc_owned()` setter so the Drop impl can tell whether the
38//! Doc or the caller owns the C allocation. Without that, every
39//! detach-and-drop path in LaTeXML Post needs this wrapper.
40
41use std::mem::ManuallyDrop;
42
43use libxml::tree::Node;
44
45/// A handle to a libxml `Node` that has been detached from its parent
46/// but whose storage is still owned by the enclosing `Document`.
47///
48/// Dropping a `DocOwnedNode` does **not** call `xmlFreeNode` on the
49/// inner C node; see the module-level docs for the rationale.
50pub struct DocOwnedNode {
51  inner: ManuallyDrop<Node>,
52}
53
54impl DocOwnedNode {
55  /// Wrap a `Node` whose C memory is owned by the enclosing `Document`.
56  ///
57  /// Typical call: `DocOwnedNode::new(xmath.clone())` right after
58  /// `doc.remove_nodes(&[xmath])`. The caller's original `Node` binding
59  /// is kept alive only for the vector's lifetime; the `DocOwnedNode`
60  /// preserves the Rc for the rest of program execution.
61  pub fn new(node: Node) -> Self { Self { inner: ManuallyDrop::new(node) } }
62
63  /// Borrow the underlying `Node`. Safe because `DocOwnedNode` keeps
64  /// the Rc alive.
65  pub fn as_node(&self) -> &Node { &self.inner }
66}
67
68impl Drop for DocOwnedNode {
69  fn drop(&mut self) {
70    // `ManuallyDrop` suppresses the inner Rc's Drop, which in turn
71    // suppresses `_Node::drop` (and the offending `xmlFreeNode` call
72    // on the unlinked subtree). The Rc control block itself leaks
73    // (small, bounded), but the underlying libxml storage is reclaimed
74    // by the enclosing `xmlFreeDoc` at Document drop time.
75  }
76}

78#[cfg(test)]
79mod tests {
80  use libxml::parser::Parser;
81
82  use super::*;
83
84  #[test]
85  fn detached_node_drop_is_noop() {
86    let parser = Parser::default();
87    let doc = parser.parse_string("<root><child/></root>").expect("parse");
88    let root = doc.get_root_element().expect("root");
89    let child = root.get_first_child().expect("child");
90    let mut child_mut = child.clone();
91    child_mut.unlink_node();
92    // Wrap and drop — must not call xmlFreeNode.
93    let wrapper = DocOwnedNode::new(child);
94    drop(wrapper);
95    // If the wrapper's drop had fired xmlFreeNode, the subsequent
96    // Document drop would double-free; the test would crash.
97    drop(doc);
98  }
99}
