1//! BibTeX.pool.ltxml — bibliographic-entry processing for LaTeXML.
2//!
3//! Perl: `LaTeXML/blib/lib/LaTeXML/Engine/BibTeX.pool.ltxml`. Loaded
4//! via `LoadPool('BibTeX')` (e.g. from `amsrefs.sty.ltxml`) or as a
5//! preload when the conversion mode is BibTeX
6//! (`Common/Config.pm:406`: `unshift(... 'BibTeX.pool')`).
7//!
8//! Status: **skeleton only**. The Perl pool is 956 lines of
9//! BibTeX-specific definitions (`\bib`, entry-type constructors,
10//! field handlers, BibKey normalization, etc.). They are not yet
11//! ported to Rust. This binding exists so every Perl pool has a
12//! 1:1 Rust counterpart per the
13//! [pool parity audit](../../../docs/POOL_PARITY_AUDIT.md), and it
14//! mirrors Perl `BibTeX.pool.ltxml` L19 (`LoadPool('LaTeX')`) so
15//! that loading `BibTeX` correctly chains in the LaTeX format.
16//!
17//! TODO: port BibTeX-specific definitions from
18//! `BibTeX.pool.ltxml`. Approximate scope:
19//! - `\bib` / `\bibitem` family (Perl ~L80-200)
20//! - Bib entry-type constructors (`@article`, `@book`, ... ~L220-500)
21//! - Field handlers and helper functions (`CleanBibKey`, `NormalizeBibKey`, `ProcessBibTeXEntry` —
22//!   currently stubbed in `latexml_package::package::amsrefs_sty`).
23//! - BibTeX special-character handling (~L800-956).
24
