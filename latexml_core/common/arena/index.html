An arena for interning strings (global, mutable, single thread only)

Note: works under the assumption of a single threaded, short-lived process, where memory starvation will not be an issue. It would need a hard reset if the same process does multiple conversions, and a different implementation if one needs a thread-local arena.

§Borrow safety

All arena access goes through with_arena_mut(), which acquires a mutable borrow from RefCell on first call and caches the raw pointer in a thread-local Cell. Re-entrant calls (e.g., pin() inside a with() closure) reuse the cached pointer without touching RefCell, eliminating borrow conflicts entirely. A RAII guard clears the pointer on scope exit (including panics), so the invariant holds through unwinds.
