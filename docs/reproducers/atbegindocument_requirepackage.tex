% Minimal reproducer: a package deferred to \AtBeginDocument must still be
% loadable — real latex.ltx disables the \@onlypreamble commands (\@preamblecmds,
% \document L54) AFTER firing the begindocument hook (L44), so \RequirePackage /
% \usepackage inside \AtBeginDocument is LEGAL.
%
% Ground truth (same host):
%   pdflatex  -> exit 0, no error
%   latexml (Perl, /usr/local/bin) -> exit 0, "No obvious problems"
%   latexml_oxide (pre-fix)        -> Error: '\RequirePackage' can only appear in the preamble
%
% Corpus witnesses: arXiv:2605.00022, arXiv:2605.00119 — both \usepackage{inconsolata},
% whose inconsolata.sty does \AtBeginDocument{...\usepackage{upquote}} → upquote.sty's
% top-level \RequirePackage{textcomp}, firing the guard at begin-document.
%
% Regression origin: upstream LaTeXML PR #2846 fixed #2754 (see
% atbegindocument_paragraph_break.tex) by clearing `inPreamble` BEFORE the hook, but
% `inPreamble` also gates the onlyPreamble guard, so it disabled \RequirePackage too
% early. Direction B decouples the guard onto its own `inBeginDocumentHook` signal:
% inPreamble clears before the hook (keeps #2754 fixed) while the guard stays
% satisfied through the hook window. Body-level \RequirePackage must STILL error
% (that IS onlypreamble; parity with pdflatex+Perl).
\documentclass{article}
\AtBeginDocument{\RequirePackage{xcolor}}
\begin{document}

\end{document}
