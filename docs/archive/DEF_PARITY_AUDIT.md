## Triage notes

- **DefPrimitiveI → DefMacro (46 total across engine+package)**. Perl's
  `DefPrimitiveI` is the literal-tokens form for one-shot atomic
  commands (typographic accents, pictures, single-character ops).
  Rust's `DefMacro!` is a tokens-expansion form. Converting these to
  `DefPrimitive!` restores the single-pass semantics.

- **DefKeyVal → DefMacro (45 in package)**. Perl's `DefKeyVal` declares a
  keyval option group that is consumed by `\setkeys`. Rust's `DefMacro!`
  treats it as a plain macro. These need real `DefKeyVal!` bindings
  (port the keyval system if it's not yet ported).

- **DefMacro → DefPrimitive (47 total)**. Reverse direction: Perl
  expects tokens-expansion, Rust uses atomic primitive. Often a
  performance choice that loses semantic fidelity (no pre-expansion
  visibility to the gullet).

- **DefConstructor → DefMacro (38 total)**. Perl emits an XML element
  directly; Rust produces tokens that expand to the element later.
  Usually functionally equivalent, but `DefConstructor!` is closer to
  Perl and correct when the element should appear verbatim at that
  point (e.g., `\left`, `\right`, picture drawing commands).

## Notes from attempted fixes

### Accented-letter cluster (`\OE \oe \AE \ae \AA \aa \O \o \L \l \ss \dh \DH
\dj \DJ \ng \NG \th \TH`)

Perl: `DefPrimitiveI('\OE', undef, "\x{0152}", robust => 1)` — robust primitive
emitting the Unicode char at stomach level. Rust currently uses a two-layer
`DefMacro!("\\lx@utf@OE", None, "\u{0152}", alias => "\\OE")` plus
`DefMacro!("\\OE", None, "\\lx@utf@OE")`.

A naïve conversion to `DefPrimitive!("\\OE", "\u{0152}", robust => true)`
broke the `lettercase_test` and `textcase_test` regression tests because
the Rust `lx_read_and_change_case` case-mapping code path assumes the
`\@uclclist` pairs `Expand!` down to *character tokens*. When the members
become primitives, `Expand!` stops at the CS level and the mapping table
ends up keyed by `"\\ae "` instead of `"æ "`, while
`lx_read_and_change_case` itself did not look up the CS-keyed entries.
Adding a raw-CS branch to `lx_read_and_change_case` was necessary but not
sufficient — also the `\lx@prepare@case@mapping` assumption that
`Expand!(\@uclclist)` returns chars breaks.

To land this cluster cleanly:
1. Change `\lx@prepare@case@mapping` to expect CS tokens (mirror Perl's
   `lookupDefinition->getExpansion->unlist`, i.e. shallow expansion only).
2. Extend `lx_read_and_change_case` to look up the CS-keyed mapping for
   bare CS tokens.
3. Convert the 19 `DefMacro!`/`lx@utf@` pairs to single-line
   `DefPrimitive!` calls.

Until that three-step landing is done, the mismatch stays. Reverted in
this session.

### B1 detailed design (current batch)

Root cause found by Perl source reading: `DefPrimitiveI(..., robust => 1)` in
`Package.pm:1286` calls `defRobustCS` which creates TWO bindings — `\cs` as an
Expandable wrapper expanding to `\protect<cs-munged>` and `<cs-munged>` (space-
suffixed) as the actual Primitive. `\MakeUppercase{\ae}` runs `\edef` which
produces `\protect \ae ` tokens; the case-changer recognizes the `\protect <cs>`
pattern and looks up in `text_uppercase`.

Three coordinated Rust edits:

1. `\lx@prepare@case@mapping` — stop using `Expand!(\@uclclist)` (deep expand;
   unfolds robust wrappers to `\protect <cs>` pairs, shifting indices). Read the
   raw expansion body via `lookup_definition_stored` + `Stored::Expandable`
   `get_expansion()`.
2. `lx_read_and_change_case` protect-branch — fix key format: mapping is stored
   with `<cs> ` (trailing space via `format!("{} ", s)`) but lookup strips the
   trailing space. Normalize lookup to the same `<cs> ` format for
   text_uppercase/text_lowercase (leave text_case_exclude on bare form).
3. Convert the 19 `\OE`/`\oe`/… `DefMacro!` chains + their `\lx@utf@*` aliases
   to `DefPrimitive!(..., robust => true)` matching Perl's `DefPrimitiveI(...,
   robust => 1)` exactly.

Verification gate: `cargo test -p latexml --test 10_expansion lettercase_test
utflettercase_test` + full workspace tests. Regression signals: any drift in
`lettercase.xml`, `utflettercase.xml`, `textcase.xml` expected outputs.
