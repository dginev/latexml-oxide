* `0901.2408` — **Out of scope** (verified 2026-05-01: Perl ALSO
  produces 4 errors on full paper under documented invocation —
  same `Unexpected:_` + 3× `malformed:ltx:XMApp in <ltx:emph>`
  pattern; both engines hit the same `\emph{...$$...$$...}`
  digester limitation). Canvas inclusion of this paper is
  spurious; not a Rust regression. Mini-repro retained below for
  reference but no fix is owed under the "match Perl" mission.

  **Re-bisected min-repro (8 lines, 2026-05-01 cron iter,
  supersedes prior 5-line):**
  ```latex
  \documentclass[a4paper,10pt]{article}
  \usepackage{amsmath}
  \newenvironment{proposition}{}{}
  \begin{document}
  \begin{proposition}
  \emph{x $N$ y $$\beta \geq d_{max} \, (\tfrac{2}{M^{\ast}} + 1)$$ z $d_{sum} = a$.}
  \end{proposition}
  \end{document}
  ```
  Produces **2 errors** of the same two types as the full paper:
  (1) `Error:Unexpected:_` (subscript leak after `$$` decomposes
  in text mode); (2) `Error:malformed:ltx:XMApp in <ltx:emph>`
  (XMath emission directly into ltx:emph). The full paper has 4
  errors because lines 296-300 (the proposition env triggering this
  one min-repro) contain more inline math after the `$$`.

  **Trigger pattern**: `\emph{ ... $$ ... _ ... $$ ... $...$ ... }`
  — display math `$$ ... $$` placed inside text-mode `\emph{}`.
  In horizontal mode (inside `\emph`), `$$` decomposes into two
  consecutive `$` toggles: open inline math + immediately close
  (empty body), leaving the math body content in TEXT mode where
  `_`/`^` raise `Unexpected:_`. The trailing inline `$d_{sum}=a$`
  then opens math but its XMath output gets stuck directly into
  the ltx:emph parent because the document tree's wrapping path
  is poisoned by the prior empty-math frame.

  **Perl behavior**: paper IS in 100k_no_problems.txt canvas — Perl
  somehow handles this without errors. Either Perl's `\emph{}` digest
  enters a different mode that treats `$$` as display math (escaping
  the surrounding emph), OR Perl's empty-math handler doesn't poison
  the document tree.

  **Fix candidate**: in `\emph` constructor's mode, allow `$$` to
  escape the `\emph{}` text frame as display math (Perl seems to
  do this). Or alternatively: detect empty `$ $` inline math and
  null-op it without leaving residue. Investigation locus:
  `tex_math.rs` `$$` gate + `\emph` constructor mode in
  `latex_constructs.rs:3094-3095`. Tractable next-session work.
