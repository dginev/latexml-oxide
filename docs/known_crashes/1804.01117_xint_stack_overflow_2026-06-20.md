# Crash: 1804.01117 xint raw-load stack overflow (SIGABRT)

Date: 2026-06-20. See docs/STABILITY_WITNESSES.md Cluster F + docs/SYNC_STATUS.md
(1804.01117 reclassification) for the full analysis.

## Repro
```
# Full paper (needs the tikz+xint cumulative context — minimal \usepackage{xintexpr}
# does NOT crash, it completes with 8 errors):
cd <1804.01117 source>   # /data/arxiv/1804/1804.01117/1804.01117.zip
latexml_oxide --preload=ar5iv.sty --destination out.xml main.tex
#   → thread '<unknown>' has overflowed its stack; fatal runtime error: stack overflow, aborting (exit 134)
# Perl matched config: latexml --includestyles main.tex  → exit 0, 39 errors, 39-byte EMPTY doc
```

## Root cause: runaway native recursion (number-arg-macro chain), ~25000 deep,
overflows the 256 MB conversion-thread stack (latexml_oxide.rs:327). Faithful fix =
a Perl-$MAXSTACK-style depth guard at the read_x_token / Expandable::invoke chokepoint.
Threshold must be corpus-calibrated (see Cluster F). LOW priority (paper fails in both engines).
