BibTeX.pool.ltxml — bibliographic-entry processing for LaTeXML.

Perl: LaTeXML/blib/lib/LaTeXML/Engine/BibTeX.pool.ltxml. Loaded via LoadPool('BibTeX') (e.g. from amsrefs.sty.ltxml) or as a preload when the conversion mode is BibTeX (Common/Config.pm:406: unshift(... 'BibTeX.pool')).

§Status (2026-05-15): Phase 1 foundation

Per docs/BIBTEX_PORT_PLAN.md, the full port is split into 6 phases. This file is at Phase 1: the BibEntry data structure + current-entry tracking + field accessors are in place; no Def* bindings yet.

The Perl pool stores entries in the State Value table under keys like BIBENTRY@<normalized-key>. Rust uses a thread-local registry instead to avoid threading a custom Stored::BibEntry variant through dump_writer / dump_reader / Stored::PartialEq. (BibEntries don’t round-trip through dumps anyway — they’re created and consumed entirely within a single conversion.)

Bind‐wise still TODO (Phases 2-6):

  • \bib / \bibitem family (Perl ~L80-200)
  • Bib entry-type constructors (@article, @book, … ~L220-500)
  • Field handlers (bib@field@* family ~60 macros)
  • BibTeX special-character handling (~L800-956)
  • bibAddToContainer / processBibNameList (need Document API integration)
